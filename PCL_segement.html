Executive Summary

This project used the Franka Emika Panda arm in Gazebo to grasp a pudding box, collect voxel data on the environment, and segment the table from the other objects using Random Sample Consensus (RANSAC). The Panda arm has an Intel Realsense D435 camera in an eye-in-hand configuration that collects RGBD data on the environment.

This was a two week long project completed with a team of 2 members.

My Contribution

I primarily worked on configuring the simulation environment and MoveIt!. I assisted with the PCL segmentation and stiching voxel snapshots.

Grasping Pudding

Here, the robot arm travels to a hardcoded position, grasps the pudding box, and lifts it. To see the lift, compare the relative height of the panda arm to the top pane of Gazebo.

A custom Action Server was created with MoveIt! to command the arm to a desired pose.

Voxel Representation of RGBD Data


Raw Voxel Representation (left), Segmented Table (middle), and Segmented Pudding (right)

Utilizing ROS's ability to create nodes in either C++ or Python, the segmentation portion of code was created in C++. This is due to better support and documention for the Point Cloud Library in C++.
